# PROMPT 模板

> 用法：把本文件夹（`放进AI文件夹的/`）整个拖进 Claude Code 的工作目录，然后把下面这段「复制给 AI 的内容」复制到对话框，**只改【】里的部分**，发送即可。
> 详细操作步骤见《2-vibe-coding指南.md》。

---

## 模板 A：从零做一个新玩法

复制下面整段给 AI ⬇️

```
你是一个玩法 H5 开发助手。请阅读当前文件夹里的以下文件，并严格遵守：
- 工程规范.md（架构、埋点、目录的硬性规范）
- func.md（埋点上报接口正本）
- report.js（埋点标准实现，直接复用，不要重写）
- demo骨架/index.html（结构参照）

【玩法创意】
- 玩法名称：【例：小麦想了解你多一点】
- 玩法类型：【例：回合制问答 / 海龟汤推理 / 修仙文字游戏】
- 一句话玩法：【例：每次随机抽 5 道选择题，用户答题，每题 bot 给个性化回应，答完生成画像】
- 核心流程：【分步描述用户从进入到结束会经历什么】
- bot 是谁：【bot 名 + 人设要点，或附上人设档案文件名】
- 玩法结束时要回传给 bot 什么：【例：用户的答题记录摘要，让 bot 能延续话题】

【埋点要求】
- sceneName 定为：【例：xiaomai_icebreaking】
- 除标准 pv/exp 外，需要的点击事件：【例：start_clk 开始、submit_clk 提交、share_clk 去聊天、retry_clk 再来一次】

【内容素材】
【把题库/题目/文案贴在这里，或说明让 AI 基于人设生成。回应文案要符合 bot 语气。】

【要求】
1. 默认输出单个 index.html（遵守工程规范的单文件原则）。
2. 配置区放在文件顶部，sceneName/bot_name/题库等集中可改。
3. 埋点直接复用 report.js 的实现，关页面上报必须用 keepalive/sendBeacon。
4. content 摘要用规则拼接（参照 demo 的 buildSummaryText），纯文本、不调 LLM。
5. 移动端优先，max-width 520px 居中。
6. 做完后，用《3-上线前验收清单.md》逐条自检并告诉我结果。
```

## 模板 B：在现有玩法上改东西

复制下面整段给 AI ⬇️

```
请基于当前文件夹里的 index.html 修改，继续遵守 工程规范.md 和 func.md。

【我要改的】
【例：把题库换成黎小姐的人设回应 / 增加一个分享按钮并加 share_clk 埋点 / 结果页加一张图】

【不要动的】
- 埋点逻辑和接口字段
- 配置区的结构

改完告诉我：改了哪几处、有没有影响埋点、需不需要我重新验收。
```

## 填写小贴士

- **玩法创意写得越具体，AI 一次做对的概率越高**。模糊的「做个好玩的游戏」会让 AI 乱发挥。
- **sceneName 一旦定了就别改**——它是后台统计这个玩法所有数据的唯一标识，改了数据就断了。命名建议 `bot名_玩法类型`，如 `xiaomai_icebreaking`、`lixiaojie_haiguitang`。
- **人设一致性**：如果 bot 有人设档案，把文件也放进文件夹，并在 PROMPT 里点名「回应文案基于 XXX 人设档案」。
- **不确定要不要拆多文件**：默认让 AI 出单文件就好，规范里 AI 会自己判断。
- **每次让 AI 改完都让它跑一遍验收清单**（模板里最后一条），比你自己肉眼检查可靠。
